# Claim Boundaries

FlowMemory Warranted Agents should sound bold because the architecture is different.

It should also stay precise.

## Strong Claims

Use these:

- Generic agents make claims. FlowMemory agents can leave warranted receipts.
- Available is not enough. Warrantable is the standard.
- Agent promises become deterministic machine histories.
- PolicyCards turn user rules into portable warranties.
- FlowBond gives the agent money on the line in the local model.
- PulsePass gives users scoped private proof.
- FlowCompiler rejects impossible machine histories.

## Not Safe To Claim

Do not claim:

- custody;
- wallet enforcement;
- x402 replacement;
- fund protection;
- live Base deployment;
- production settlement;
- production adjudication;
- production verifier infrastructure;
- production private compute;
- zero-knowledge privacy in v0;
- semantic truth;
- work-quality proof;
- guaranteed fraud prevention.

## Best Public Line

```text
Generic agents make claims. FlowMemory agents can leave warranted receipts.
```

## Best Technical Line

```text
FlowMemory Warranted Agents is a local deterministic framework for turning agent promises into receipt-bound machine histories with bond-modeled settlement and scoped private proof.
```
